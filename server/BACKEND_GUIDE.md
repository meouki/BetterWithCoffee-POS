# Better with Coffee — Backend Integration Guide

Hello! This guide provides the exact technical specifications and JSON payloads the frontend expects. Use this to design your database and API endpoints.

## 1. Project Configuration
- **Base URL**: The frontend expects the backend at `http://localhost:5000` (configurable in `.env`).
- **Image Path**: Save product images to `public/uploads/` and serve them statically.

## 2. API Entities & JSON Schemas

### A. Product
Used in POS for ordering and Dashboard for management.
```json
{
  "id": 1,
  "name": "Caramel Macchiato",
  "category": "Hot Drinks",
  "base_price": 180.00,
  "is_available": true,
  "image_url": "http://localhost:5000/uploads/macchiato.webp",
  "modifiers": true
}
```
*Note: Category must exactly be one of: `Cold Drinks`, `Hot Drinks`, `Blended Drinks`, `Frappe Drinks`, `Pastries`.*

### B. Order
Submitted when a sale is finalized.
```json
{
  "id": "ORD-123456",
  "timestamp": "2024-03-01T10:00:00Z",
  "items": [
    {
      "id": 1,
      "name": "Caramel Macchiato",
      "quantity": 2,
      "snapshot_price": 180.00
    }
  ],
  "subtotal": 360.00,
  "vat": 43.20,
  "total": 403.20,
  "payment_method": "Cash",
  "order_type": "Dine-In",
  "cashier": "Admin User"
}
```

### C. System Log (Notification)
Created automatically on Sales or Menu edits.
```json
{
  "id": "1709740000000",
  "timestamp": "2024-03-01T10:01:00Z",
  "type": "SALE", 
  "message": "New Sale: ORD-123456",
  "details": "Total: ₱403.20",
  "cashier": "Admin User"
}
```
*Types: `SALE`, `MENU_EDIT`, `ALERT`.*

### D. Inventory Item
Tracks raw materials and packaging.
```json
{
  "id": 1,
  "name": "Espresso Beans",
  "category": "Raw Materials",
  "stock": 12.00,
  "unit": "kg",
  "threshold": 5.00,
  "last_updated": "2024-03-01T08:30:00Z"
}
```

## 3. Mandatory Endpoints

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| **GET** | `/api/products` | Get menu items |
| **POST** | `/api/products` | Add new item (with `multipart/form-data` for images) |
| **PATCH** | `/api/products/:id` | Update price/availability |
| **POST** | `/api/orders` | Submit transaction |
| **GET** | `/api/orders` | Fetch full history for reports |
| **GET** | `/api/inventory` | Get current stock levels |
| **PATCH** | `/api/inventory/:id` | Adjust stock/threshold |
| **POST** | `/api/logs` | Append to notification text file |
| **GET** | `/api/logs` | Read contents of notification text file |
| **DELETE** | `/api/logs` | Wipe/Clear text file |

## 4. Image Processing
The POS must remain fast. When receiving a `POST /api/products`:
1. Use `sharp` to convert images to `.webp`.
2. Resize to a max width of `400px`.
3. Save with a clean, unique filename.

## 5. User Roles (Reference)
- `Master`: Full access (Users/Products/Reports)
- `Admin`: View Reports/Dashboard only
- `Cashier`: Access the POS screen only
